# 安全政策

## 报告漏洞

如果您发现安全漏洞，请通过以下方式报告：

1. **GitHub Issues**：[提交安全漏洞](https://github.com/your-username/ai-red-team-skills-manager/security/advisories/new)
2. **Email**：security@example.com

请不要通过公开的 GitHub Issues 报告安全漏洞。

## 安全漏洞处理流程

1. 收到漏洞报告后，我们将在 24 小时内确认
2. 我们将评估漏洞的严重程度
3. 我们将开发修复方案
4. 我们将发布安全更新
5. 我们将公开披露漏洞（在修复后）

## 支持的版本

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 安全最佳实践

### 1. 安装 Skills 时

- 只从可信来源安装 Skills
- 安装前检查 Skills 的安全性
- 定期更新已安装的 Skills

### 2. 使用 Skills 时

- 不要在生产环境中测试 Skills
- 定期审计 Skills 的安全性
- 及时删除不再使用的 Skills

### 3. 配置系统时

- 使用强密码
- 启用双因素认证
- 定期备份配置

## 安全审计

我们定期进行安全审计，包括：

- 代码审查
- 依赖项检查
- 漏洞扫描
- 渗透测试
